Request & Data Flow

1
Browser navigates to any route
Every request hits proxy.ts (Next.js middleware). It calls updateSession() which refreshes the Supabase JWT cookie. If no valid session exists the request is redirected to /auth/login. Authenticated users visiting /auth/* are redirected to /lists.
2
Server Component renders the page
lists/page.tsx or lists/[id]/page.tsx runs on the server. It creates a cookie-based Supabase client (lib/supabase/server.ts), queries the database (RLS automatically filters rows the user may access), and returns fully-rendered HTML with initial data baked in.
3
User triggers a mutation
Client component (e.g. ItemList) calls a Server Action (e.g. addItem()). The server action runs with full server privileges, creates a Supabase server client, writes to the database, then calls revalidatePath() to invalidate the Next.js cache for that route. The client applies an optimistic update immediately so the UI feels instant, and rolls back on error.
4
Realtime sync (shared lists)
ItemList subscribes to Supabase Realtime on mount via the browser client (lib/supabase/client.ts). It listens for postgres_changes on the items table filtered by list_id. When another user adds, checks, or deletes an item, the event arrives over WebSocket and setItems() updates local React state — no page refresh required.
5
Autocomplete
lists/[id]/page.tsx fetches the user's user_item_history (ordered by use_count desc, limited to 200) at render time and passes it as a suggestions prop to ItemList. As the user types, the list is filtered client-side — no extra network call. Each successful item insert causes the bump_item_history DB trigger to upsert that item name into user_item_history, incrementing the use count.

Source File Reference

Config & Database
FileTypePurpose
supabase/migrations/0001_init.sql DB Entire database schema in one migration. Creates all 4 tables, indexes, RLS policies for every operation, the bump_item_history trigger (autocomplete), has_list_access() helper (used inside RLS to avoid recursion), find_user_by_email() RPC (invite flow), and enables Realtime on items, lists, and list_members.
next.config.ts Config Next.js configuration. Currently empty — no custom options needed.
tsconfig.json Config TypeScript configuration. Defines the @/ path alias pointing to src/.
postcss.config.mjs Config Enables Tailwind CSS via PostCSS.
Middleware & Supabase Clients
FileTypePurpose
src/proxy.ts Middleware Next.js middleware entry point. Matches all non-static routes and delegates to updateSession(). Acts as the application's authentication gate on every request.
src/lib/supabase/middleware.ts Middleware updateSession() — refreshes the Supabase JWT cookie on every request to keep sessions alive. Enforces route rules: unauthenticated users are redirected to /auth/login; authenticated users on /auth/* are redirected to /lists.
src/lib/supabase/server.ts Server Factory for the server-side Supabase client. Reads and writes the JWT via Next.js cookies(). Used by all Server Components and Server Actions.
src/lib/supabase/client.ts Client Factory for the browser-side Supabase client. Used only by ItemList to open a Realtime WebSocket subscription.
src/lib/types.ts Types Shared TypeScript interfaces for List, ListMember, and Item. Single source of truth for data shapes used across client and server code.
App Shell
FileTypePurpose
src/app/layout.tsx Server Root HTML layout. Sets page title and description metadata, loads Geist Sans and Geist Mono fonts, and wraps all pages in a full-height <body>.
src/app/page.tsx Server Root route /. Contains a single redirect('/lists') — the real entry point is the lists page.
src/app/globals.css CSS Imports Tailwind base, components, and utilities. Applied globally via layout.tsx.
Auth Feature — /auth
FileTypePurpose
src/app/auth/actions.ts Action Server actions: login calls signInWithPassword, signup calls signUp, logout calls signOut and redirects to login. All return { error } on failure.
src/app/auth/AuthForm.tsx Client Shared email/password form used by both login and signup pages. Accepts a mode prop to switch labels and actions. Manages loading and inline error state.
src/app/auth/login/page.tsx Server Login page. Renders <AuthForm mode="login" />.
src/app/auth/signup/page.tsx Server Signup page. Renders <AuthForm mode="signup" />.
Lists Feature — /lists
FileTypePurpose
src/app/lists/page.tsx Server Lists index page. Fetches all accessible lists, splits into "My lists" (owner) and "Shared with me" (member) sections. Renders CreateListForm and a DeleteListButton per owned list.
src/app/lists/actions.ts Action Server actions: createList inserts a new list owned by the current user. deleteList removes it. inviteMember resolves an email to a user ID via the find_user_by_email RPC then inserts into list_members. leaveList deletes the caller's membership row.
src/app/lists/CreateListForm.tsx Client Collapsible form to create a new list. Starts as a "+ New list" dashed button; expands to show a name input and a "Shared list" checkbox. Calls the createList server action on submit.
src/app/lists/DeleteListButton.tsx Client Renders an × button per list. On first click it switches to inline "Delete / Cancel" confirmation, requiring a second click before calling deleteList — prevents accidental deletes.
List Detail Feature — /lists/[id]
FileTypePurpose
src/app/lists/[id]/page.tsx Server List detail page. Fetches the list, its items (ascending by created_at), and the user's item history for autocomplete (top 200 by use_count). Passes data as props to ItemList. Shows InviteForm only for the list owner on shared lists; shows LeaveListButton for non-owners.
src/app/lists/[id]/actions.ts Action Server actions: addItem inserts a new item (triggering the history upsert). toggleItem flips is_checked. deleteCheckedItems bulk-deletes all checked items for the list in one query.
src/app/lists/[id]/ItemList.tsx Client The most complex component. Manages local item state. For shared lists, subscribes to Supabase Realtime on mount and handles INSERT/UPDATE/DELETE postgres_changes to keep state in sync across all connected users. Implements keyboard-navigable autocomplete (arrow keys, Enter, Escape) filtered client-side from the suggestions prop. All mutations use optimistic updates that roll back on error.
src/app/lists/[id]/InviteForm.tsx Client Email input that calls the inviteMember server action. Displays inline success ("Invited!") or error feedback. Only rendered for the list owner on shared lists.
src/app/lists/[id]/LeaveListButton.tsx Client "Leave" button for non-owners of a shared list. Shows a confirm() dialog before calling leaveList, then navigates back to /lists. Only rendered when the current user is not the owner.
