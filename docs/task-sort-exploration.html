Task lists — sort views & the view switcher

Two new things for the task-list detail screen: a switcher between Manual (added order / hand-reordered, drag grips) and By date, and the by-date view itself — tasks grouped into Overdue · Today · Tomorrow · <weekday> · Later · No date. The Done section stays at the bottom in both. Below: switcher placements first, then three takes on the grouped view. Colours reuse the app's due palette (red overdue, amber today/soon, grey later).

1 · The view switcher

Where it sits and how it reads. It lives just under the add-task field, above the list. Manual shows drag grips; Date hides them (order is the date).

S1 — segmented control

recommended
A full-width two-up pill. Unmistakable, thumb-friendly, both options always visible. Most "iOS settings" familiar.

2 · The by-date view

Same data, grouped by when it's due. Overdue is pinned on top in red. Within days 2–6 each weekday is its own section (e.g. "Wednesday"); a week out or more collapses to "Later"; undated tasks sink to "No date". The three takes differ in how loud the section headers are and whether rows still show a date pill.

D1 — quiet labels

recommended
Lightweight uppercase labels + count, like the existing swimlane style. Overdue/Today get a colour; the rest are grey. Row-level due pills are dropped — the section already says when. Calmest, most list-like.

Switcher — reading the three

  • S1 segmented: clearest, both modes visible, biggest tap target. Costs a full row of vertical space.
  • S2 icon toggle: tiny, pairs with a task-count line. Current mode is implied by the icon — least discoverable.
  • S3 dropdown: text-labelled and future-proof (room for more sort modes), but hides options behind a tap.

By-date view — reading the three

  • D1 quiet labels reuses the existing swimlane style; dropping the row pill keeps it calm and scannable.
  • D2 bars chunk hardest — great for a glance, but lots of colour on a long chore list.
  • D3 spine + dates keeps the exact date per row; most precise, slightly noisier. Worth it only if exact dates matter.
  • All three: undated tasks sink to "No date"; only non-empty sections render so a short list stays short.
My lean: S1 (segmented) + D1 (quiet labels). The segmented control is the most obvious "two ways to look at this", and quiet labels keep the grouped view feeling like the same calm checklist — the colour earns its place only on Overdue/Today. If exact dates turn out to matter in use, D1 → D3 is a small, additive change (just re-show the row pill). The switcher's "Manual" mode keeps the drag grips; "By date" hides them. Section labels follow the task UI's English (Today / Tomorrow / Wednesday / Later / No date).
