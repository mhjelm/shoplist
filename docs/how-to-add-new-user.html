How to add a new user

Shoplist is invitation-only. Public signup is disabled in Supabase — only users you explicitly invite can create an account.

Invite a family member

  1. Open the Supabase dashboard and pick the Shoplist project.
  2. Go to Authentication → Users.
  3. Click Add user → Send invitation.
  4. Enter the person's email address and confirm.
  5. They receive an email with a magic link. Clicking it lets them set a password and lands them in the app at /lists.

The invited email becomes a regular auth.users row, so RLS, list sharing and find_user_by_email all work the same as for any other user.

If the invite email never arrives

  • Check the spam folder.
  • In the dashboard, the user shows up in Authentication → Users with status Invited. You can resend the invite from the row's ⋯ menu.
  • Supabase's default invite template is in English. Customise it under Authentication → Email Templates → Invite user if you want Swedish copy.

Sharing a list with the invited user

Once they've accepted the invite and logged in at least once, you can share a list with them from inside the app:

  1. Open the list → tap the pencil/edit button in the header to enter edit mode.
  2. Scroll to the Dela med section and type their email (or pick from the chips of previously-invited emails).
  3. Submit. They'll see the list under Delade listor on their /lists page.

How to re-enable public (self-)signup

⚠ Only do this if you genuinely want anyone on the internet to be able to register

Shoplist has no signup gating in app code — authorization comes from Supabase + RLS. The moment self-signup is on, anyone who knows the app URL can create an account (they won't see anyone else's lists, but they will have a working empty account on your project, count against your Supabase quotas, and be able to invite themselves into the user table).

  1. Open the Supabase dashboard → Authentication → Sign In / Providers (older dashboards: Authentication → Settings).
  2. Find the Email provider.
  3. Turn on "Allow new users to sign up" (sometimes labelled Enable email signup / Enable signups).
  4. Save.

The /auth/signup page in the app already exists and will start working again immediately — no code changes or redeploy required.

Optional hardening for public-signup mode

  • Enable Confirm email under the Email provider so new accounts must click a link before they can log in.
  • Turn on Supabase's CAPTCHA protection (hCaptcha or Turnstile) to slow bot signups.
  • Consider adding rate-limits under Authentication → Rate Limits.

How to revoke access

  1. In Authentication → Users, find the user and use the row's ⋯ menu → Delete user. This cascades through RLS: their list_members rows disappear and any lists they owned are deleted along with their items.
  2. Alternatively, to keep their account but kick them out of a single list, open the list in edit mode and remove them from the Dela med section.
