Mixed (A / B)

  • One place — no tab to remember, recency sort just works.
  • Tiny code delta: today's two sections stay; rows gain an icon + kind marker.
  • Visual noise — every row must declare its kind or it's ambiguous.
  • "Shared with me" already adds a section; a third axis (kind) competing for the same rows can feel busy once lists pile up.

Separated (A / B)

  • Clean mental model — groceries and chores never blur together.
  • Create field + add-flow can be kind-specific (no mode toggle in the input).
  • Tabs (B) hide one kind and need remembered state + a cross-over NEW signal.
  • Sections (A) push shopping down once you have several task lists, or vice-versa.

My lean: Separated · A (grouped sections) for v1 — closest to today's structure, no hidden state, and the create field can default its kind by which section you're nearest. Revisit tabs only if one kind routinely grows past a screenful.
